# ТЗ для помощника — обход листингов Anjuna + конкурентов (браузером)

**Цель:** (1) точная перепись — где реально выставлены виллы Anjuna и сколько юнитов;
(2) факты для сравнительной оценки против 14 конкурентов «глазами гостя».
**Время:** ~4–5 часов. **Файлы для заполнения:** `listing_census.csv` и `listing_scorecard.csv`
(вкладки в нашем Google Sheet). **Скриншоты** складывай в папку `screenshots/` с именами по шаблону
`anjuna-b1_airbnb_D1.png`, `coz_booking_grid.png`.

## 0. Подготовка (10 мин)
- Браузер в **режиме инкогнито** (чтобы не влияла история), язык **English**, валюта **USD**.
- VPN не включать (смотрим как гость «по умолчанию»; если ты в Индонезии — ок, но отметь это в заметках).
- Открой обе таблицы + эту инструкцию.

## Тестовые даты (единые для ВСЕХ — иначе цены несравнимы)
- **D1 (межсезонье):** заезд **17 авг 2026 (пн) → выезд 20 авг 2026** · 3 ночи · **2 взрослых**
- **D2 (пик):** заезд **21 дек 2026 → выезд 28 дек 2026** · 7 ночей · **4 взрослых**
Если на дату юнит занят — попробуй сдвинуть на ±1 неделю и пометь «(сдвиг)». Фиксируй **итоговую
цену на чекауте** (со сборами и налогами), а не цену «от».

## Часть 1 — Перепись Anjuna (~90 мин) → `listing_census.csv`
Правило: юнит считается «живым», только если доходишь до последнего шага чекаута с датами D1.
Агрегатные листинги («Anjuna 2BR Villas») обязательно **открывать и считать номера внутри**.

1. **Airbnb — главный трюк:** открой любой листинг Anjuna из таблицы (напр. B1) → клик по имени
   хоста → **View profile** → появится СПИСОК ВСЕХ его объектов. Найди в нём все «Anjuna» —
   это почти полная перепись Airbnb за 10 минут. Скриншот профиля целиком (прокруткой).
   Для каждого юнита: URL, BR, цена D1 итоговая, рейтинг, число отзывов, instant book (молния)?,
   мин. ночей, число фото (открой галерею — счётчик «1/NN»).
2. **Booking.com:** поиск «Anjuna» + Pecatu, даты D1. Открой каждый наш объект; в агрегатах —
   в таблице номеров посчитай, сколько отдельных вилл внутри и сколько доступно на D1.
3. **Agoda, Trip.com, Traveloka, Expedia, Vrbo, TripAdvisor:** поиск «Anjuna» + Uluwatu на каждом.
   Нашёл — строка в census; не нашёл — напиши «не найдено (дата, поиск)» в строке канала.
4. **Marriott Homes & Villas:** поиск Uluwatu.
5. **Прямой сайт:** balinestvillas.guestybookings.com — пролистай ВЕСЬ список объектов, выпиши все Anjuna.
6. **Google:** «Anjuna villa Uluwatu» → вкладки Maps и Travel. Есть ли карточка отеля/виллы? Скриншот.

## Часть 2 — «Как ищет гость» (~30 мин)
Три поиска в инкогнито, всё скриншоть (сетку выдачи!):
1. Airbnb: «Uluwatu», даты D1, 2 взрослых → листай выдачу. **На какой позиции первый Anjuna?**
   Кто стоит ДО него (названия первых 10)?
2. Booking: «Uluwatu», D1, 2 взрослых, фильтр «Villas» → то же: позиция Anjuna, кто выше.
3. Google: «villa uluwatu private pool beach access» → что в топ-10 и в блоке Google Travel? Anjuna есть?
Запиши позиции в заметку `search_positions.md` (просто списком).

## Часть 3 — Факты для скоркарты (~2–2.5 ч) → `listing_scorecard.csv`
Для **каждого живого юнита Anjuna** и **каждого из 14 конкурентов** (они уже вписаны в таблицу)
зафиксируй на даты **D1** (одной строкой, колонки уже есть):
- итоговая цена D1 (и D2, если успеваешь — через слэш: «$520 / $980»)
- отмена: бесплатная до …? / non-refundable
- instant book (Y/N) · мин. ночей · депозит/правила, если бросаются в глаза
- **число фото** (галерея «1/NN») · есть ли подписи к фото (Y/N)
- заголовок листинга — скопируй дословно
- первые 2 строки описания — скопируй дословно
- рейтинг · число отзывов · бейджи (Superhost / Guest favorite / Preferred)
- отвечает ли менеджмент на отзывы (просмотри 5 свежих) Y/N
- скриншоты: hero-фото + шапка листинга каждого объекта

**Баллы (1–7 блоки) сама НЕ ставь** — только факты и скрины. Баллы выставим вторым проходом по
анкорам (лучший в сете = верх шкалы), чтобы оценка была защищаемой.

## Правила
- Всё в одинаковых условиях: те же даты, та же валюта, инкогнито.
- Ничего не бронировать, никому не писать. Это только просмотр.
- Если страница ведёт себя странно (другая цена при повторном заходе) — скриншот обоих вариантов + пометка.
- Не оценивай «нравится/не нравится» — только факты. Вкус обсудим на скоринге.
